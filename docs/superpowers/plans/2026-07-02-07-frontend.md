# Plan 7/8: Frontend (React + Vite + TypeScript) — Implementation Plan

> **For the implementer (Cursor):** Work task-by-task, commit per task. Frontend lives in `frontend/`; commands run there (`npm run dev|build|test`). Python suite must stay green (`.venv/bin/pytest -q` from repo root at the end).
>
> **Contract check before starting** — backend must be Plan-6 complete; verify endpoints exist:
> ```
> grep -r "api/study/curriculum\|api/chat\|api/quiz/next\|api/admin/profiles\|api/corpus/sources\|api/memory\|api/exports" mslearn/server/routers | wc -l   # expect > 0 for each
> ```

**Goal:** The six-view localhost UI: Corpus manager, Curriculum browser, Teaching view (markdown + tension callouts + citations), Quiz, streaming Chat, Memory panel — plus an admin bar (profile toggle, spend, tunables link) and exports button.

**Architecture:** `frontend/` Vite + React 18 + TypeScript. `src/api/client.ts` (typed fetch wrapper + SSE reader), `src/views/*` one component per view, `react-router-dom` routing, minimal hand-rolled CSS (`src/app.css`) — no UI framework (YAGNI). Dev: Vite proxy `/api` → `http://localhost:8000`. Prod: `npm run build` → `frontend/dist` served by FastAPI (Plan 6 static mount). Tests: Vitest + @testing-library/react, `fetch` mocked; SSE reader unit-tested against a scripted ReadableStream.

## Docs to read before starting

| Topic | URL | Need |
|---|---|---|
| Vite proxy | https://vitejs.dev/config/server-options.html#server-proxy | `/api` proxy config |
| Fetch streaming | https://developer.mozilla.org/en-US/docs/Web/API/ReadableStream (reader pattern) | parse `data: {...}\n\n` frames from `response.body.getReader()` (EventSource can't POST — use fetch streaming) |
| react-markdown | https://github.com/remarkjs/react-markdown | render teach markdown |
| Vitest + RTL | https://vitest.dev/guide/ + https://testing-library.com/docs/react-testing-library/intro/ | jsdom env, `vi.stubGlobal("fetch", ...)` |

## Global Constraints

- TypeScript strict; all API payload types in `src/api/types.ts` mirroring backend JSON exactly (write them from the Plan-6 router code, not guesses)
- No UI framework/deps beyond: react, react-dom, react-router-dom, react-markdown; dev: vite, typescript, vitest, jsdom, @testing-library/react, @testing-library/user-event, @vitejs/plugin-react
- Chat uses fetch-streaming SSE parsing (POST body needed); deltas render incrementally; citations chips rendered from the final `done` frame
- Teaching view renders `## Where sources disagree` prominently (styled callout box — CSS class `.tension`), citations `[claim:x]` rendered as superscript chips that expand locator info from the concept payload
- Every view handles loading + error states; API errors surface the backend `detail` string
- `npm test` green, `npm run build` clean, python suite untouched-green; commit per task

## Tasks

### Task 1: Scaffold + API client + routing shell
`frontend/` via `npm create vite@latest frontend -- --template react-ts` (pin react-router-dom, react-markdown; add vitest/RTL dev deps); `vite.config.ts` with `/api` proxy + vitest jsdom config; `src/api/types.ts` (SourceRow, ConceptMeta, ConceptDetail, TeachResponse, QuizNext, QuizGrade, ChatFrame, MemoryItem, ProfileInfo, TunableRow, SpendSummary — typed from Plan-6 routers); `src/api/client.ts`: `api<T>(path, init?) -> Promise<T>` (JSON, throws `ApiError(detail, status)`), `streamChat(question, sessionId, onDelta, onDone)` (fetch POST, getReader loop, frame parser tolerating partial chunks — buffer split on `\n\n`, JSON.parse each `data: ` payload); `App.tsx` router shell with nav (Corpus / Curriculum / Quiz / Chat / Memory) + `AdminBar` placeholder; `app.css` basics.
**Tests:** frame parser unit tests (split-across-chunks case, done frame); `api()` error propagation.
**Commit:** `feat(ui): scaffold, typed client, SSE frame parser, routing shell`

### Task 2: AdminBar + Corpus manager
`AdminBar`: profile dropdown (GET/POST profiles), spend summary (total cost + calls, refresh 30s), exports button (POST /api/exports kinds all → list file paths in a toast/dialog). `CorpusView`: add-source form (ref, role select, type select optional, local checkbox), sources table (title/ref, role, status, progress `done+failed/total`, pause/resume buttons), domain-profile selector, "Run synthesis" button.
**Tests:** profile switch calls POST and re-renders active; add-source posts payload and prepends row; pause button hits endpoint; error banner on 422.
**Commit:** `feat(ui): admin bar and corpus manager`

### Task 3: Curriculum + Teaching views
`CurriculumView`: ordered concept list (order_index), name + summary + conflict-count badge; click → `/concepts/:id`. `ConceptView`: fetches detail + teach markdown (`react-markdown`); regenerate button (`?force=1`); `.tension` styling for the disagree section (wrap via markdown component override on h2 text match or CSS `:has` fallback — simplest: custom h2 renderer that adds the class and wraps following siblings is overkill; acceptable: style the whole section by rendering markdown split at the heading — implement `splitTeachMarkdown(md)` helper returning `{main, tension}` and render tension in a styled box; unit-test the splitter); citation chips: regex `[claim:x]` → superscript with tooltip from detail payload's citations; per-claim flag button (POST flag with reason prompt → refetch).
**Tests:** splitter (with/without tension section); citation chip rendering count; flag flow calls endpoint.
**Commit:** `feat(ui): curriculum browser and teaching view with tension callouts + citations`

### Task 4: Quiz view
`QuizView`: GET next → show question; textarea answer; submit → grade card (correct badge, score, explanation markdown with citation chips); "next question" loop; stats table (per concept: attempts, avg score).
**Tests:** answer submit posts + renders grade; next fetch after grade; stats render.
**Commit:** `feat(ui): quiz flow with graded feedback`

### Task 5: Chat + Memory views + build integration
`ChatView`: message list, input; `streamChat` appends deltas live to the pending assistant bubble; done frame renders citation chips; session id in `sessionStorage`; error frame handling. `MemoryView`: list items grouped by category, delete buttons, 503 state ("memory unavailable — mem0 not installed"). Build integration: `make ui-build` target (root Makefile: `cd frontend && npm run build`), README frontend section; verify FastAPI serves `frontend/dist` (manual step note).
**Tests:** chat streams two deltas then done (scripted ReadableStream through mocked fetch) and renders incrementally; memory delete removes row; 503 renders empty-state.
**Commit:** `feat(ui): streaming chat and memory panel; build integration`
